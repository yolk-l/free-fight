# v7 玩法设计：房间制地下城

## 设计背景

v6 的单张大地图 + 迷雾探索目标性不足，玩家随意放置怪物也能推进。v7 改为以撒式房间结构：预生成树状房间地图，每个房间内有事件格，玩家在事件格上部署怪物，击杀后触发格子效果。所有事件清除后出口开启，玩家点击出口进入下一房间。

## 核心约束（不变）

- 玩家唯一操作 = 从 3 张候选选 1 张拖到事件格上
- 无主动技能、无弃牌、无装备
- 怪物只由玩家部署产生（Boss 召唤除外）
- 线框 Demo 定位

---

## 1. 核心循环

```
玩家看到3张候选 → 选1张拖到当前房间的事件格上
→ 怪物出现在该格子（固定不动）
→ 英雄沿 A* 最短路径走向最近怪物
→ 到达攻击范围，进入战斗（实时近战）
→ 击杀后英雄走到怪物所在格子，触发该格子效果
→ 该事件格标记为已清除
→ 英雄寻找场上下一个最近怪物，继续走
→ 场上无怪时英雄原地等待
→ 2s CD后刷新3张新候选
→ 所有事件格清除 → 出口开启
→ 玩家点击出口 → 英雄走到出口 → 进入下一房间
→ 循环直到击杀 Boss 或英雄死亡
```

**策略核心**：玩家通过"在哪个事件格放怪物"决定获得哪些增益/承受哪些代价，以及英雄的战斗顺序。

## 2. 地图结构

### 2.1 树状房间拓扑

- **房间数**：6-9 个（`GameConfig.ROOM_COUNT_MIN` ~ `ROOM_COUNT_MAX`）
- **拓扑**：树形连接，随机生成父节点
- **房间类型**：起始 / 普通 / 宝藏 / 危险 / 精英 / Boss
- **Boss 房间**：树中最深的叶节点
- **无迷雾**：房间内所有格子直接可见

### 2.2 单房间布局

- **尺寸**：10×8 格（`GameConfig.ROOM_W` × `ROOM_H`）
- **每格大小**：64×64 像素
- **结构**：墙壁围边，内部大片空地，0-2 个柱子（单格墙壁）
- **出生点**：房间中央区域
- **出口**：位于房间边缘中间位置，指向树中相邻房间
- **通往已清除房间的出口**：自动关闭（变为墙壁），防止回头

### 2.3 出口交互

- 出口在事件未全部清除时**不可通行**（暗色显示）
- 所有事件清除后出口**开启**（亮绿色 + 动画）
- 玩家**点击出口**后英雄走向出口，到达后自动切换房间
- 通往精英房间的出口显示橙色**"精英"**标记
- 通往 Boss 房间的出口显示红色**"BOSS"**标记

### 2.4 相机

- 跟随英雄，平滑插值
- 玩家可右键/中键拖拽、滚轮缩放（0.6x ~ 1.5x）
- 右上角小地图显示房间树结构 + 当前房间高亮

## 3. 事件格

### 3.1 总览

| 类别 | 格子 | 视觉 | 击杀怪物后触发 |
|------|------|------|---------------|
| **空地** | 普通地面 | 浅灰 | — |
| **正面** | 治愈泉 | 浅蓝 + | 回复 10 HP |
| **正面** | 力量祭坛 | 红 A | 攻击+2（持续3次击杀） |
| **正面** | 铁壁祭坛 | 银灰 D | 防御+2（持续3次击杀） |
| **正面** | 共鸣水晶 | 紫 R | 下次击杀共鸣进度 ×2 |
| **正面** | 宝箱 | 金 T | 随机永久属性 +1 |
| **负面** | 毒沼 | 暗绿 P | -8 HP + 毒涂层×2 |
| **负面** | 陷阱 | 暗红 ! | -12 HP + 攻速+0.5 |
| **负面** | 诅咒地 | 暗紫 C | 攻击-2 + 共鸣×2 |
| **负面** | 减速泥 | 棕 ~ | 移速减半 + 防御+3 |
| **随机** | 问号格 | 黄 ? | 从事件池随机抽取 |
| **特殊** | 出口 | 灰/绿 > | 清除所有事件后开启 |
| **特殊** | 起点 | 绿 S | 英雄出生位置 |

**负面事件附带补偿**：负面格不是纯惩罚，而是"代价换收益"的风险收益权衡。

### 3.2 连锁倍率 + 怪物-格子亲和

#### 连锁倍率

房间内维护"连锁计数器"，每触发一个格子效果 +1。格子效果数值按连锁数缩放：

`效果 = 基础值 × (1.0 + 连锁数 × 0.3)`

| 连锁 | 倍率 |
|------|------|
| 0 | ×1.0 |
| 1 | ×1.3 |
| 2 | ×1.6 |
| 3 | ×1.9 |
| 4 | ×2.2 |
| 5+ | ×2.5+ |

**策略**：负面格尽量先清（低倍率少吃亏），正面格留到后面清（高倍率多拿收益）。玩家通过部署位置控制英雄的清除顺序（英雄走向最近怪物）。切换房间后连锁重置。

#### 怪物-格子亲和

每个怪物有亲和属性，每个事件格有亲和类型。匹配时效果 ×1.5。

| 亲和 | 怪物 | 匹配格子 |
|------|------|----------|
| **猛攻** | 狼、哥布林、螳螂 | 力量祭坛、陷阱 |
| **坚韧** | 骷髅、石像鬼 | 铁壁祭坛、减速泥 |
| **灵巧** | 蝙蝠、毒蛇、萤火虫 | 共鸣水晶、诅咒地、问号格 |
| **生命** | 史莱姆、树人 | 治愈泉、毒沼、宝箱 |

**组合公式**：`效果 = 基础值 × 连锁倍率 × 亲和倍率`

- 数值型效果（HP、攻击、防御等）按倍率缩放
- 标志型效果（共鸣×2标记、减速倍率、Buff持续次数）不缩放

卡牌 UI 显示亲和标签，格子上显示亲和标签，拖拽时匹配的格子显示 "×1.5!" 提示。

### 3.3 房间事件构成（固定比例 + 随机填充）

每种房间保证正面/负面/随机的最低比例，剩余位从混合池随机填充：

| 房间类型 | 正面 | 负面 | 问号 | 总数 |
|---------|------|------|------|------|
| 起始 | 3 | 1 | 1 | 5-8 |
| 普通 | 2 | 2 | 1 | 5-8 |
| 宝藏 | 4 | 1 | 0 | 5-8 |
| 危险 | 1 | 3 | 1 | 5-8 |
| 精英 | 2 | 2 | 1 | 5-8 |

总事件数 = `randi_range(EVENTS_PER_ROOM_MIN, EVENTS_PER_ROOM_MAX)` = 5~8。
保证每个房间都有正面和负面选项，确保策略选择。

### 3.4 部署规则

- 只能在**未清除的事件格**上部署怪物
- 不能在空地、墙壁、出口、已清除的事件格上部署
- 效果仅在**击杀怪物后英雄站上该格时**触发（路过不触发）
- 触发后事件格变为空地（一次性效果）

## 4. 英雄移动

### 4.1 寻路

- A* 寻路（Godot AStar2D），四方向移动（无对角）
- 逐格移动，每步 0.3 秒（`Hero.GRID_STEP_TIME`）
- 到达攻击范围（48px）内开始攻击

### 4.2 目标选择

- 英雄走向场上最近的怪物（A* 路径距离）
- 击杀后：英雄走到怪物所在格子 → 触发效果 → 切换下一目标
- 场上无怪：英雄原地等待
- 出口开启后：玩家点击出口，英雄走向出口

---

## 5. Run 结构

### 5.1 一局流程

1. 生成树状房间地图（6-9 个房间）
2. 英雄从起始房间（room 0）出发
3. 清除每个房间的所有事件 → 出口开启 → 选择出口进入下一房间
4. 到达 Boss 房间 → Boss 直接生成 → 击杀 Boss → 通关
5. 英雄 HP ≤ 0 → 失败

### 5.2 难度缩放

基于房间在树中的深度：

| 深度 | 倍率 |
|------|------|
| 0-1 | ×1.0 |
| 2-3 | ×1.3 |
| 4-5 | ×1.6 |
| 6+ | ×2.0 |
| Boss | ×2.5 |

### 5.3 Boss

- Boss 位于树中最深的叶节点房间
- Boss 房间无事件格，进入后直接生成 Boss
- Boss 可召唤小怪（出现在房间空格上）
- 击杀 Boss → 通关

## 6. 保留的系统

### 6.1 候选/手牌
- 3 选 1 + 2s CD + 全手牌刷新
- NextHandPreview：CD 期间显示下一批
- 卡池权重

### 6.2 怪物机制

| 怪物 | 机制 |
|------|------|
| 史莱姆 | 死亡分裂：小史莱姆出现在周围空格 |
| 蝙蝠 | 飞行 |
| 狼 | 群族：范围内有同族 → 攻速+50% |
| 哥布林 | 死亡爆炸：范围内所有单位受伤 |
| 骷髅 | 墓碑复活 |
| 石像鬼 | 防御光环：范围内友方怪+2防 |
| 毒蛇 | 死亡毒池：所在格变为毒沼（永久） |

### 6.3 共鸣/被动
- 7 路径 × 3 阶 = 21 个被动技能
- 任意两路径 Tier I 解锁 8 个混合机制

### 6.4 Combo
- 生态专精 / 异种联动 / 密集部署

### 6.5 击杀属性
- 击杀累计属性（按怪种映射 attack/defense/attack_speed/armor_penetration/max_hp）
- 精英变体 ×2

## 7. 与 v6 的主要差异

| v6 | v7 |
|----|----|
| 单张 30×20 大地图 | 树状 6-9 个 10×8 房间 |
| 迷雾覆盖未探索区域 | 无迷雾，房间内全可见 |
| 英雄路过格子触发效果 | 击杀怪物后站上格子才触发 |
| 可在任意空地部署 | 只能在事件格上部署 |
| 视野塔/传送阵 | 已移除 |
| 距离起点的格数决定难度 | 房间深度决定难度 |
| 无明确房间概念 | 清除事件 → 开出口 → 选路 |
