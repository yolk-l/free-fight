# v2 实现计划：手牌持仓压力与部署策略

> 依据：[06-v2-手牌持仓压力设计.md](./06-v2-手牌持仓压力设计.md)

> **范围**：v2 核心（持仓负面 + 弃牌 + 可读 UI）已落地；掉落分化、落点预览等见 [05-依赖与扩展.md](./05-依赖与扩展.md) backlog。

## 目标与验收

**目标**：玩家因「留牌有战斗耦合代价」而必须管理部署/弃牌；部署=结束该卡持仓税 + 承担场上风险。

**验收标准**：

1. 手牌中每张怪卡有可见「拿着」负面；手牌区有合计 debuff 与失血速率。
2. 英雄 `get_combat_stats()` 正确反映叠加后的攻/防/攻速；失血按秒扣 HP。
3. 拖放部署或弃牌后，该卡负面立即从合计中消失。
4. 弃牌：冷却结束可弃 1 张，无代价，局内次数不限；冷却中 UI 明确。
5. 三只怪物 `.tres` 持仓负面符合物种分工表（史莱姆轻、哥布林含失血）。
6. 更新 `development-scope.mdc` 与知识目录；无自动刷怪、无部署格。

### 阶段 1：数据与配置（约 0.5 天）

| 任务 | 文件 | 说明 |
|------|------|------|
| 扩展 `MonsterData` | `data/monster_data.gd` | `@export var hold_penalty: HoldPenaltyStats`；`hold_bleed_per_sec` |
| 常量 | `data/game_config.gd` | `DISCARD_COOLDOWN_SEC := 10.0`（见 [08-v2-数值初稿.md](./08-v2-数值初稿.md)） |
| 配置表 | `resources/monsters/*.tres` | 见 **08 数值初稿** §3、§9 |
| 英雄基准 | `resources/hero_default.tres` | **120 HP**（08 建议） |
| 规则 | `.cursor/rules/development-scope.mdc` | 允许：持仓负面、即时制弃牌；仍禁止自动刷怪/部署格 |

**持仓叠加结构**（新建或放在 `card_hand.gd`）：

```gdscript
# 伪代码：get_hold_penalty_sum() -> CombatStats + bleed_total
for id in _card_ids:
    sum += monster_data.hold_penalty
    bleed += monster_data.hold_bleed_per_sec
```

### 阶段 2：持仓结算接入战斗（约 1 天）

| 任务 | 文件 | 说明 |
|------|------|------|
| `CardHand.get_hold_penalty_sum()` | `scripts/battle/card_hand.gd` | 遍历 `_card_ids` 线性叠加，无 clamp |
| 失血 tick | `card_hand.gd` 或 `battle_controller.gd` | 每秒对 Hero `take_damage` 或直改 `base_stats.hp`（推荐直改 hp 并 `stats_changed`，避免触发死亡误判时用 `mini`） |
| `Hero.get_combat_stats()` | `scripts/battle/hero.gd` | 顺序：基础 → 装备 `apply_bonus` → 减去持仓 sum；`hp` 仍用 `base_stats.hp` |
| 部署/拾牌 | 已有 `consume_card` / `add_card` | 确保 `_rebuild_ui` 后合计刷新 |
| 信号 | `CardHand` | 可选 `hold_penalty_changed`，供 Hero UI 刷新 |

**注意**：失血与「攻降为 0」叠加时可能极难；阶段 5 再调 `.tres` 与 `DISCARD_COOLDOWN_SEC`。

---

### 阶段 3：弃牌（即时制冷却）（约 0.5～1 天）

| 任务 | 文件 | 说明 |
|------|------|------|
| 冷却状态 | `battle_controller.gd` | `_discard_cooldown: float`，`_physics_process` 中 `-= delta`；就绪时 ≥0 |
| 弃牌 API | `card_hand.gd` | `discard_card(monster_id)`：与 `consume_card` 相同移除逻辑，不部署 |
| 入口 UI | `battle_scene.tscn` + 新脚本或 `card_hand` | 选手牌 +「弃牌」按钮；仅冷却就绪可点 |
| 弃牌后 | `battle_controller` | 调用 `discard_card` → 重置 `_discard_cooldown = DISCARD_COOLDOWN_SEC` |
| 无代价 | — | 不扣血、不加 debuff |

**交互草案**：

- 手牌某张选中时，侧栏或卡上显示「弃」（冷却中灰显 + 倒计时秒数）。
- 也可右键手牌弃牌（与按钮二选一，避免两套都做）。

### 阶段 4：UI 可读（约 0.5～1 天）

| 任务 | 位置 | 说明 |
|------|------|------|

### 阶段 5：验收与 playtest 调参（约 0.5 天）

| 项 | 初版默认值 | 调参方向 |
|----|------------|----------|
| `DISCARD_COOLDOWN_SEC` | **10** | 太难则 8 |
| 哥布林失血 | **0.6/s** | 太强则 0.4 |
| 英雄 HP | **120** | 太难则 140 |
| 手牌上限 | 7（不变） | 满手过难可降到 5～6 |
| 叠加上限 | 无 | 过难再加 soft cap（另开任务） |

**手动测试清单**（v2 上线时已验收，playtest 仍可按此回归）：

- [x] 仅留史莱姆：英雄防/攻略降，无失血
- [x] 留 2 张哥布林：攻防暴降 + 持续失血，部署一只后该卡负面消失
- [x] 冷却中无法弃；冷却好弃牌无扣血，负面立即减少
- [x] 装备加成与持仓负面同时存在，UI 数值正确

## 风险与对策

| 风险 | 对策 |
|------|------|
| 无叠加上限 + 7 张手牌 → 英雄瞬间废 | 阶段 5 调低单卡负面；必要时 hotfix 加 cap |
| 失血 + 多怪围攻暴毙 | 与部署风险叠加；先调失血与冷却，再考虑小幅加血 |
| `get_combat_stats` 顺序错误 | 单元场景测试：仅手牌 debuff、仅装备、两者皆有 |
