# 数据层

## Resource 类型

| 类 | 脚本 | 用途 |
|----|------|------|
| `CombatStats` | `data/combat_stats.gd` | 攻 / 最大 HP / 当前 HP / 防 / 攻速（单位基准、装备加成） |
| `BuffDef` | `data/buff_def.gd` | Buff/Debuff 定义模板：id、持续类型、叠加上限、通用 KV modifiers |
| `BuffInstance` | `data/buff_instance.gd` | 运行时 Buff 实例（RefCounted）：层数、剩余时间/次数、来源标识 |
| `MonsterData` | `data/monster_data.gd` | 怪物模板：id、名称、基础属性、线框色、移速、`hold_debuff: BuffDef` |
| `EquipmentData` | `data/equipment_data.gd` | 装备模板：id、名称、槽位类型、属性加成 |
| `EquipmentQuality` | `data/equipment_quality.gd` | 品质枚举（白/绿/蓝/紫）+ 倍率/颜色/权重 + `roll_quality()` |
| `EquipmentAffix` | `data/equipment_affix.gd` | 前缀词缀枚举 + 适用槽位 + 属性加成 + `roll_affix(slot)` |
| `EquipmentInstance` | `data/equipment_instance.gd` | 运行时装备实例（RefCounted）：base_data + quality + affix → 计算最终属性 |

### 装备品质与词缀

掉落时为每件装备随机生成品质和前缀，通过 `EquipmentInstance` 包装：

**品质等级**（必有）：

| 品质 | 颜色 | 属性倍率 | 掉落权重 |
|------|------|----------|----------|
| 普通 | 白 | 1.0x | 50 |
| 优秀 | 绿 | 1.3x | 30 |
| 稀有 | 蓝 | 1.6x | 15 |
| 史诗 | 紫 | 2.0x | 5 |

**前缀词缀**（50% 概率无，50% 随机1个适用词缀）：

| 前缀 | 效果 | 适用 |
|------|------|------|
| 锋利的 | +3 攻 | 武器 |
| 迅捷的 | +0.3 攻速 | 武器 |
| 坚韧的 | +2 防 | 护甲 |
| 坚固的 | +20 max_hp | 护甲 |

**属性计算**：`floor(基础属性 × 品质倍率) + 前缀加成`

**运行时模型**：`EquipmentInventory.slots` 存 `EquipmentInstance`（非 `EquipmentData`）。

### EquipmentBackpack

`scripts/battle/equipment_backpack.gd`：战斗中的装备背包（8 格）。

- `add_item(instance)` — 背包满时丢弃最早获得的（index 0）
- `remove_item(index)` — 取出并返回
- `get_item(index)` / `get_count()` / `is_full()`
- 信号 `backpack_changed` — 通知 UI 刷新

### CombatStats 关键方法

- `duplicate_stats()`：复制一份运行时属性。
- `get_attack_interval()`：`1.0 / attack_speed`，用于普攻 CD。
- `apply_bonus(bonus)`：将加成叠加到副本上；**Hero** 在 `get_combat_stats()` 中使用。
- `zero_bonus()`：全 0 加成，供 `EquipmentInventory` 聚合初值。

### BuffDef

Buff/Debuff 的静态定义（Resource），可作为 `.tres` 配置：

| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | `StringName` | 唯一标识 |
| `display_name` | `String` | 显示名 |
| `is_debuff` | `bool` | 是否为负面效果 |
| `duration_type` | `DurationType` | `PERMANENT`（条件移除）/ `TIMED`（秒数倒计时）/ `COUNTED`（事件次数） |
| `duration_sec` | `float` | TIMED 时的持续秒数 |
| `duration_count` | `int` | COUNTED 时的触发次数 |
| `trigger_event` | `StringName` | COUNTED 时的消耗事件名（如 `on_attack`、`on_hit`） |
| `max_stacks` | `int` | 最大叠加层数 |
| `modifiers` | `Dictionary` | 通用属性修改 KV，如 `{"attack": -2, "defense": -1, "bleed_per_sec": 0.6}` |

### BuffInstance

运行时实例，由 `BuffContainer.add_buff()` 创建：

- `def: BuffDef` — 指向定义
- `stacks: int` — 当前层数
- `remaining_sec` / `remaining_count` — 剩余倒计时
- `source_id: StringName` — 来源标识，用于条件移除（如 `hold_slime_0`）
- `get_modifier(key)` — 返回 `def.modifiers[key] * stacks`

### 持仓负面（Buff 系统实现）

| 阶段 | 行为 |
|------|------|
| 添加手牌 | CardHand 为每张卡在 Hero 的 BuffContainer 上添加一个 PERMANENT debuff 实例 |
| 效果叠加 | BuffContainer 汇总所有实例的 modifiers（线性求和） |
| 英雄生效 | `Hero.get_combat_stats()` 叠加 buff modifiers 后 clamp 下限 |
| 移除 | 部署/弃牌时通过 `remove_buff_by_source(source_id)` 精确移除 |

### 伤害公式（`CombatUnit.try_attack`）

```
damage = max(1, 攻击方.attack - 防守方.defense)
```

## 游戏常量

| 类 | 脚本 | 常量 |
|----|------|------|
| `GameConfig` | `data/game_config.gd` | `ATTACK_RANGE = 48`（普攻距离，像素） |
| | | `DISCARD_COOLDOWN_SEC = 10.0`（弃牌冷却） |
| | | `MIN_ATTACK = 1`、`MIN_DEFENSE = 0`、`MIN_ATTACK_SPEED = 0.4`（buff 叠加后的下限） |

## 配置实例目录

```
resources/
├── hero_default.tres      # 英雄初始 CombatStats（120 HP）
├── monsters/
│   ├── slime.tres         # hold_debuff → buffs/hold_slime.tres
│   ├── wolf.tres
│   └── goblin.tres
├── equipment/
│   ├── wood_sword.tres    # slot_type: weapon
│   └── leather_armor.tres # slot_type: armor
└── buffs/
    ├── hold_slime.tres    # PERMANENT debuff: defense -1
    ├── hold_wolf.tres     # PERMANENT debuff: attack -2
    └── hold_goblin.tres   # PERMANENT debuff: attack -2, defense -1, bleed_per_sec 0.6
```

新增内容：**仅加 `.tres`**，保持 `id` 字段唯一；`DataRegistry` 启动时自动加载怪物和装备。

## DataRegistry

- 扫描 `resources/monsters/`、`resources/equipment/` 下所有 `.tres`。
- 以资源的 `id: StringName` 为键存入字典。
- 对外 API：
  - `get_monster(id)` / `get_equipment(id)`
  - `get_all_monster_ids()` / `get_all_equipment_ids()` → `Array[StringName]`
  - `get_random_monster_id()`：手牌初始发牌、装备掉落兜底用。
