# 数据层

## Resource 类型

| 类 | 脚本 | 用途 |
|----|------|------|
| `CombatStats` | `data/combat_stats.gd` | 攻 / 最大 HP / 当前 HP / 防 / 攻速（单位基准、装备加成） |
| `HoldPenaltyStats` | `data/hold_penalty_stats.gd` | 手牌持仓负面：攻 / 防 / 攻速增量，**字段默认 0** |
| `MonsterData` | `data/monster_data.gd` | 怪物模板：id、名称、基础属性、线框色、移速、`hold_penalty`、`hold_bleed_per_sec` |
| `EquipmentData` | `data/equipment_data.gd` | 装备模板：id、名称、槽位类型、属性加成 |

### CombatStats 关键方法

- `duplicate_stats()`：复制一份运行时属性。
- `get_attack_interval()`：`1.0 / attack_speed`，用于普攻 CD。
- `apply_bonus(bonus)`：将加成叠加到副本上；**Hero** 在 `get_combat_stats()` 中使用。
- `apply_penalty(penalty)`：将持仓合计（`CombatStats` 副本）加到攻/防/攻速上，并做**属性下限** clamp（见 `GameConfig`）。
- `zero_bonus()`：全 0 加成，供 `EquipmentInventory` 聚合与 `CardHand.get_hold_penalty_sum()` 初值。

> **勿用 `CombatStats` 作为 `hold_penalty` 子资源**：未显式字段会继承 `attack=10` 等默认值，叠加后会把惩罚变成加成。见 [08-v2-数值初稿.md](./08-v2-数值初稿.md) §9。

### HoldPenaltyStats

- 字段：`attack`、`defense`、`attack_speed`，默认均为 **0**。
- `merge_into(sum: CombatStats)`：累加进手牌持仓合计（由 `CardHand.get_hold_penalty_sum()` 调用）。

### 持仓负面：叠加 vs 生效

| 阶段 | 行为 |
|------|------|
| 手牌合计 | 每张卡的 `HoldPenaltyStats` **线性求和**，合计过程不 clamp |
| 英雄生效 | `Hero.get_combat_stats()` 经 `apply_penalty()` 后：`attack ≥ MIN_ATTACK`，`defense ≥ MIN_DEFENSE`，`attack_speed ≥ MIN_ATTACK_SPEED` |

### 伤害公式（`CombatUnit.try_attack`）

```
damage = max(1, 攻击方.attack - 防守方.defense)
```

## 游戏常量

| 类 | 脚本 | 常量 |
|----|------|------|
| `GameConfig` | `data/game_config.gd` | `ATTACK_RANGE = 48`（普攻距离，像素） |
| | | `DISCARD_COOLDOWN_SEC = 10.0`（弃牌冷却） |
| | | `MIN_ATTACK = 1`、`MIN_DEFENSE = 0`、`MIN_ATTACK_SPEED = 0.4`（持仓负面后的下限） |
| `GameIds` | `data/game_ids.gd` | 怪物 / 装备 id 常量（可选引用） |

> 视觉常量（颜色、字号）在 `scripts/ui/wireframe_theme.gd`，不含玩法数值。

## 配置实例目录

```
resources/
├── hero_default.tres      # 英雄初始 CombatStats（v2：120 HP）
├── monsters/
│   ├── slime.tres         # hold_penalty 子资源脚本 = HoldPenaltyStats
│   ├── wolf.tres
│   └── goblin.tres
└── equipment/
    ├── wood_sword.tres    # slot_type: weapon
    └── leather_armor.tres # slot_type: armor
```

新增内容：**仅加 `.tres`**，保持 `id` 字段唯一；`DataRegistry` 启动时自动加载。

## DataRegistry

- 扫描 `resources/monsters/`、`resources/equipment/` 下所有 `.tres`。
- 以资源的 `id: StringName` 为键存入字典。
- 对外 API：
  - `get_monster(id)` / `get_equipment(id)`
  - `get_all_monster_ids()` / `get_all_equipment_ids()` → `Array[StringName]`
  - `get_random_monster_id()`：手牌初始发牌、装备掉落兜底用。
