# 场景与 UI

## 场景列表

| 场景 | 路径 | 脚本 | 功能 |
|------|------|------|------|
| 主菜单 | `scenes/main_menu.tscn` | `scripts/main_menu.gd` | 开始游戏、进入图鉴 |
| 图鉴 | `scenes/codex/codex_scene.tscn` | `scripts/codex_scene.gd` | 展示已解锁怪物 / 装备 |
| 战斗 | `scenes/battle/battle_scene.tscn` | `scripts/battle/battle_controller.gd` | 核心玩法 |

## 子场景（战斗用）

| 预制 | 路径 | 根脚本 |
|------|------|--------|
| 英雄单位 | `scenes/battle/hero_unit.tscn` | `hero.gd` + 子节点 `EquipmentInventory` |
| 怪物单位 | `scenes/battle/monster_unit.tscn` | `monster.gd` |
| 属性条 | `scenes/battle/ui/stat_bar.tscn` | `stat_bar.gd` |
| 掉落物 | `scenes/battle/loot_drop.tscn` | `loot_drop.gd` |

### 单位节点结构

```
Hero / Monster (Node2D)
├── Body (Sprite2D)       # assets/hero.png 或 assets/monsters/{id}.png
├── NameLabel (Label)
└── StatBar (实例)        # ProgressBar + ATK/DEF/HP/SPD 文本
```

手牌卡由代码动态创建 `MonsterCardUI`（`PanelContainer` + 怪物图标 + 持仓 hint）。

## 资源目录（`assets/`）

| 路径 | 用途 |
|------|------|
| `assets/hero.png` | 英雄精灵 |
| `assets/monsters/{id}.png` | 怪物精灵、手牌图标、卡牌掉落图标 |
| `assets/equipment/{id}.png` | 装备栏与装备掉落 |
| `assets/ui/*.png` | 战斗背景、底栏、部署区框、菜单背景等 |

缺图时：怪物仍可用 `MonsterData.wireframe_color` 给卡面/面板上色；单位 Body 可能无纹理。

维护脚本：`generate_ui_assets.py`（根目录，生成 `assets/ui/` 占位图，可选）。

## UI 风格

- **玩法定位**：线框 Demo（无剧情/音效/正式美术包）；当前已用简单贴图 + `wireframe_theme` 配色。
- 常量：`scripts/ui/wireframe_theme.gd`（`BG`、`PANEL`、`BORDER`、`ACCENT` 等）。
- 按钮：`scripts/ui/wireframe_button.gd` → 主菜单 / 图鉴 / Game Over。

### 战斗 UI 分区

| 区域 | 节点 | 说明 |
|------|------|------|
| 顶栏左 | `UI/TopBar/TopLabel` | 操作说明（拖拽、持仓、弃牌） |
| 顶栏右 | `UI/TopBar/SurvivalLabel` | 存活秒数 |
| 左侧框 | `UI/HeroZoneFrame` | 英雄区域示意 |
| 右侧 | `UI/BattlefieldDropZone` | 拖放部署区（拖入时高亮） |
| 底图 | `UI/BottomBg` | 底栏背景（`mouse_filter = IGNORE`） |
| 底栏 | `UI/BottomPanel` | 见下方 VBox 顺序 |
| 居中弹层 | `UI/GameOverPanel` | 失败面板 |

### BottomPanel（VBox 自上而下）

1. `EquipmentBar` — 武器 / 护甲槽（图标 + 名称）
2. `CardHand` — 标题 Label + 动态手牌卡
3. `HoldSummary` — 持仓合计文案
4. `DiscardRow` — `BtnDiscard` + `DiscardCooldown`

## 图鉴逻辑

`codex_scene.gd`：

- 遍历 `DataRegistry` 全部 id。
- `monster_id in GameManager.codex_unlocked_monsters` → 显示名称，否则 `???`。
- 装备同理。

## 场景切换 API（GameManager）

```gdscript
go_to_main_menu()  # res://scenes/main_menu.tscn
go_to_battle()     # res://scenes/battle/battle_scene.tscn
go_to_codex()      # res://scenes/codex/codex_scene.tscn
```

`change_scene_to_file` 会销毁当前场景树，战斗内状态（怪物列表等）不保留；图鉴解锁列表在 Autoload 中跨场景保留。
