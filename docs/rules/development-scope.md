# FreeFight 开发范围（禁止自由扩展）

本规则约束本仓库内一切开发与 AI 辅助改动。**未在下方「允许范围」中列出的内容，默认不得实现**；用户明确要求或更新本规则文件后方可扩展。

## 必须遵守

1. **先问再做**：需求含糊、超出允许范围、或会改变核心玩法时，先向用户确认，不要自行加功能。
2. **最小改动**：只改与当前任务直接相关的文件；不顺手重构、不顺手「顺便优化」无关模块。
3. **不扩计划外内容**：不要添加用户未要求的系统（例：技能树、关卡编辑器、联网、存档、剧情、音效、正式美术资源）。
4. **保持线框 Demo 定位**：UI 继续用 `ColorRect` / `Panel` / `Label` 等线框表现；不引入外部美术包替代整体风格，除非用户明确要求。
5. **架构约定不可擅自推翻**：
   - 战斗单位：`CombatUnit` 薄基类；`Hero` 含被动技能状态字段；`Monster` 含 7 种独特机制。
   - 配置用 `Resource`（`MonsterData` / `CombatStats`）；跨场景状态用 `GameManager` / `DataRegistry` / `RunManager` Autoload。
   - 战斗节拍：`DungeonBattleController._physics_process` → `tick_combat`、`card_hand.tick`。
6. **没有怪物自动生成**：场上怪物**只能**由玩家拖拽手牌部署产生（Boss 召唤、被动技能召唤的友方骷髅除外）；不得实现定时刷怪、波次刷怪、环境刷怪或任何不消耗手牌的自动召唤逻辑。
7. **玩家唯一操作 = 部署 + 点击出口**：除"从 3 张候选选 1 张拖到事件格"和"点击已开启的出口"外，玩家不做任何其他操作。**禁止**主动技能释放按键、抉择弹窗、弃牌按钮、装备穿卸点击等。

## 当前允许的功能范围（v7）

| 模块 | 允许内容 |
|------|----------|
| 主菜单 | 开始游戏、图鉴、场景切换 |
| 图鉴 | 仅怪物（基于 `GameManager` 解锁列表；未解锁显示 `???`） |
| 战斗 | 英雄自动巡猎（A* 走向最近怪物近战攻击）；**拖拽 3 张候选之一至事件格**部署；怪物固定不动；2s 部署 CD；CD 结束自动刷新 3 张新候选 |
| 地图 | 树状 6-9 房间（10×8 格/房间），房间类型：起始/普通/宝藏/危险/精英/Boss；固定比例事件构成 + 随机填充 |
| 房间流程 | 清除所有事件格 → 出口开启 → 玩家点击出口 → 英雄走到出口 → 进入下一房间；已清除房间出口自动关闭 |
| 事件格 | 5 种正面（治愈泉/力量祭坛/铁壁祭坛/共鸣水晶/宝箱）+ 4 种负面（毒沼/陷阱/诅咒地/减速泥）+ 问号格；击杀怪物后站上格子触发；负面格附带补偿效果 |
| 候选卡 | 加权随机 3 张不重复；每张显示机制简述 + 共鸣进度；10% 概率为精英版（金边 + HP/攻 +30%） |
| Combo | 模式识别（生态专精：同种连铺 3 / 异种联动：5 个 2-pair / 密集部署：4s 内 4 只） |
| 共鸣 | 7 路径 × 3 阶 = 21 个被动技能；任意两路径 Tier I 解锁 8 个混合机制 |
| 英雄 | 150 HP / 10 攻 / 3 防 / 1.0 攻速 / 48 近战射程；逐格移动 0.3s/步；被动技能状态 |
| 怪物 | 仅玩家部署（Boss 召唤、友方骷髅除外）；7 种独特机制（分裂/飞行/群族/爆炸/墓碑/光环/毒池） |
| 击杀属性 | 自动累计（按怪种映射 attack/defense/attack_speed/armor_penetration/max_hp） |
| 难度缩放 | 按房间深度：0-1×1.0 / 2-3×1.3 / 4-5×1.6 / 6+×2.0 / Boss×2.5 |
| Boss | 4 个 Boss，位于树中最深叶节点房间，进入后直接生成；击杀 Boss 即通关 |
| 失败 | 英雄 HP ≤ 0 → Run 失败；可重新开始或回主菜单 |

## 明确禁止（除非用户更新本规则）

- **玩家主动操作**（除部署和点击出口外）：技能按键、抉择弹窗、弃牌按钮、装备穿卸点击、英雄移动指令
- 装备系统（已删除）
- 持仓 debuff（已删除）
- 弃牌冷却按钮、新卡冷却
- 迷雾系统、视野塔、传送阵
- 怪物自动生成（除 Boss 召唤、骷髅被动召唤外）
- 元素克制、复杂 AI、技能树、关卡编辑器、波次胜利条件、多人、存档、音效、正式美术资源
- 大规模重命名目录、替换引擎版本、引入新框架

## 已移除机制（勿恢复）

用户已明确要求去掉，**不得在无指示情况下加回**：

- `DeploySlot` / 部署格交互（v1 移除）
- `MonsterSpawner` / 自动刷怪（v1 移除）
- 装备系统（v5 移除）
- 持仓 debuff（v5 移除）
- 远近部署分区（v5 移除）
- 旧 10 个固定 Combo 配方（v5 改为模式识别）
- 主动弃牌按钮 / 新卡冷却（v5 移除）
- 迷雾 / 视野塔 / 传送阵（v7 移除）
- 单张大地图探索（v7 改为房间制）
- 路过触发格子效果（v7 改为击杀触发）
- Voronoi 地形 / MapTerrainType / TerrainCell（v6 移除）

## 数据与内容扩展

- **可加**：在 `resources/monsters/` 增加 `.tres` 配置（数值、名称、颜色），同步在 `RunManager.KILL_STAT_GAIN` 加映射，在 `CardHand.MECHANIC_SHORT` 加机制简述，在 `Monster._apply_mechanic_flags` 加机制 flag。
- **慎加**：新脚本、新 Autoload、新场景——须与上表「允许范围」一致，且改动面尽量小。

## 与用户协作

- 若实现与口头需求冲突，**以用户最新说明 + 本规则为准**。
