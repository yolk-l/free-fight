# FreeFight 开发范围（禁止自由扩展）

本规则约束本仓库内一切开发与 AI 辅助改动。**未在下方「允许范围」中列出的内容，默认不得实现**；用户明确要求或更新本规则文件后方可扩展。

## 必须遵守

1. **先问再做**：需求含糊、超出允许范围、或会改变核心玩法时，先向用户确认，不要自行加功能。
2. **最小改动**：只改与当前任务直接相关的文件；不顺手重构、不顺手「顺便优化」无关模块。
3. **不扩计划外内容**：不要添加用户未要求的系统（例：技能树、关卡编辑器、联网、存档、剧情、音效、正式美术资源）。
4. **保持线框 Demo 定位**：UI 继续用 `ColorRect` / `Panel` / `Label` 等线框表现；不引入外部美术包替代整体风格，除非用户明确要求。
5. **架构约定不可擅自推翻**：
   - 战斗单位：`CombatUnit` 薄基类；`Hero` 含被动技能状态字段；`Monster` 含 7 种独特机制。
   - 配置用 `Resource`（`MonsterData` / `CombatStats`）；跨场景状态用 `GameManager` / `DataRegistry` / `RunManager` Autoload。
   - 战斗节拍：`BattleController._physics_process` → `tick_combat`、`terrain_system.process_monsters`、`card_hand.tick`。
6. **没有怪物自动生成**：场上怪物**只能**由玩家拖拽手牌部署产生（Boss 召唤、被动技能召唤的友方骷髅除外）；不得实现定时刷怪、波次刷怪、环境刷怪或任何不消耗手牌的自动召唤逻辑（含 `MonsterSpawner` 及同类系统）。
7. **玩家唯一操作 = 部署**：除"从 3 张候选选 1 张拖到战场某位置"外，玩家不做任何其他操作。**禁止**主动技能释放按键、抉择弹窗、弃牌按钮、装备穿卸点击等。

## 当前允许的功能范围（v5）

| 模块 | 允许内容 |
|------|----------|
| 主菜单 | 开始游戏、图鉴、场景切换 |
| 图鉴 | 仅怪物（基于 `GameManager` 解锁列表；未解锁显示 `???`） |
| 战斗 | 英雄自动巡猎（走向最近怪物近战攻击）；**拖拽 3 张候选之一至战场任意位置**部署；怪物固定不动；2s 部署 CD；CD 结束自动刷新 3 张新候选 |
| 地图地形 | 4 种永久地形（灵泉/水晶矿/岩地/荆棘丛），开局随机生成 3-4 个，影响部署和击杀效果 |
| 范围效果 | 6 种临时效果（共鸣祭坛/荆棘地/圣光圈/暗影域/共鸣节点/腐毒地）；由共鸣 Tier II/III 被动召唤（在击杀位置），6-8 秒后消失 |
| 候选卡 | 加权随机 3 张不重复；每张显示机制简述 + 共鸣进度；10% 概率为精英版（金边 + HP/攻 +30%） |
| Combo | 模式识别（生态专精：同种连铺 3 / 异种联动：5 个 2-pair / 密集部署：4s 内 4 只） |
| 共鸣 | 7 路径 × 3 阶 = 21 个被动技能；触发方式：事件 / 概率 / 内置 CD；任意两路径 Tier I 解锁 8 个混合机制 |
| 英雄 | 攻/防/HP/攻速/穿甲；基准 **150 HP / 10 攻 / 3 防 / 1.0 攻速 / 180 移速**；近战巡猎攻击（48 射程）；被动技能状态（暴击/护盾/闪避叠加/共生回血/毒/召唤） |
| 怪物 | 仅玩家部署（Boss 召唤、被动召唤的友方骷髅除外）；7 种独特机制（分裂/飞行/群族/爆炸/墓碑/光环/毒池） |
| 击杀属性 | 自动累计（按怪种映射 attack/defense/attack_speed/armor_penetration/max_hp） |
| Run 结构 | 4 场递进战斗（前 3 场击杀 12/18/24 + 60s 存活 + 场上无怪 / 第 4 场 Boss 战），HP/属性累计/共鸣/混合跨场保留，怪物难度倍率 1.0/1.2/1.5/2.0 |
| Boss | 4 个 Boss（铁甲巨兽/暗影刺客/腐朽巫师/亡灵领主），开局预览特性+克制提示，第 4 场 Boss 战，击杀 Boss 即通关；Boss 战不生成地形 |
| 失败 | 英雄 HP ≤ 0 → Run 失败；可重新开始或回主菜单 |

## 明确禁止（除非用户更新本规则）

- **玩家主动操作**（除部署外）：技能按键、抉择弹窗、弃牌按钮、装备穿卸点击、英雄移动指令
- 装备系统（已删除：EquipmentData / Inventory / Backpack / 词缀 / 品质 / Loot 装备掉落）
- 持仓 debuff（已删除：HoldPenaltyStats / hold_xxx.tres）
- 弃牌冷却按钮、新卡冷却
- 部署区分区（近/中/远）、点击手牌即时部署
- 怪物自动生成（除 Boss 召唤、骷髅被动召唤外）
- 元素克制、复杂 AI、技能树、关卡编辑器、波次胜利条件、多人、存档、音效、正式美术资源
- 将装备逻辑放回 `CombatUnit` 基类
- 大规模重命名目录、替换引擎版本、引入新框架

## 已移除机制（勿恢复）

用户已明确要求去掉，**不得在无指示情况下加回**：

- `DeploySlot` / 部署格交互（v1 移除）
- `MonsterSpawner` / 自动刷怪（v1 移除）
- 装备系统（v5 移除）
- 持仓 debuff（v5 移除）
- 远近部署分区（v5 移除）
- 旧 10 个固定 Combo 配方（v5 改为模式识别）
- 主动弃牌按钮 / 新卡冷却（v5 移除）

## 数据与内容扩展

- **可加**：在 `resources/monsters/` 增加 `.tres` 配置（数值、名称、颜色），同步在 `RunManager.KILL_STAT_GAIN` 加映射，在 `CardHand.MECHANIC_SHORT` 加机制简述，在 `Monster._apply_mechanic_flags` 加机制 flag。
- **慎加**：新脚本、新 Autoload、新场景——须与上表「允许范围」一致，且改动面尽量小。

## 与用户协作

- 用户说「后续会添加更多 rules」：新约束应写入 `.cursor/rules/` 新 `.mdc` 文件，或经用户同意后再改本文件。
- 若实现与口头需求冲突，**以用户最新说明 + 本规则为准**。
